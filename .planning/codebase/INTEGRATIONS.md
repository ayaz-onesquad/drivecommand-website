# External Integrations

**Analysis Date:** 2026-03-31

## APIs & External Services

**Authentication & User Management:**
- **Supabase Auth** - User registration, sign-in, MFA, OAuth
  - SDK/Client: `@supabase/supabase-js`, `@supabase/ssr`
  - Auth tokens: JWT in HTTP-only cookies (server-side via SSR)
  - Configured in: `apps/web/src/lib/supabase/{client,server,admin,middleware}.ts`, `LineUp/src/services/supabase.ts`

**AI & Document Processing:**
- **Anthropic Claude API** - Document text extraction, analysis, profit predictions
  - SDK: `@anthropic-ai/sdk` 0.77.0
  - API Key: `ANTHROPIC_API_KEY` (server-side)
  - Used in: `apps/web/src/app/(owner)/actions/ai-documents.ts`
  - Endpoint: Claude models (claude-3.5-sonnet by default, configurable)

**File Storage:**
- **AWS S3 / Cloudflare R2** - Document uploads, retrieval, deletion
  - SDK: `@aws-sdk/client-s3`, `@aws-sdk/s3-request-presigner`
  - Configuration:
    - `S3_ENDPOINT` - S3 endpoint URL (or R2 API endpoint)
    - `S3_REGION` - Region or "auto" (for Cloudflare R2)
    - `S3_ACCESS_KEY_ID` - AWS/R2 access key
    - `S3_SECRET_ACCESS_KEY` - AWS/R2 secret key
    - `S3_BUCKET` - Bucket name
  - Client: `apps/web/src/lib/storage/s3-client.ts`
  - Operations (presigned URLs):
    - GET: `GetObjectCommand` + `getSignedUrl` - Download documents
    - PUT: `PutObjectCommand` + `getSignedUrl` - Upload documents
    - DELETE: `DeleteObjectCommand` - Remove files
  - Used in:
    - `apps/web/src/app/(owner)/actions/{ai-documents,driver-documents,load-documents}.ts` - Upload
    - `apps/web/src/app/api/documents/upload/route.ts` - API upload
    - `apps/web/src/app/api/mobile/driver/documents/[id]/url/route.ts` - Signed download URL
    - `apps/web/src/lib/storage/{presigned,multipart}.ts` - Helper utilities

**Email & Notifications:**
- **Resend** - Transactional emails (invitations, password resets, notifications)
  - SDK: `resend` 6.9.2
  - API Key: `RESEND_API_KEY`
  - From: `RESEND_FROM_EMAIL` (must be verified in Resend dashboard)
  - Default fallback: `DriveCommand <onboarding@resend.dev>`
  - Client: `apps/web/src/lib/email/resend-client.ts`
  - Email templates: `@react-email/components` + `@react-email/render`
  - Used for: Owner invitations, tenant invitations, notifications

**Push Notifications (Mobile):**
- **Expo Push Notifications** - Native push for iOS/Android
  - SDK: `expo-server-sdk` 6.1.0, `expo-notifications` 55.0.13
  - Client: `apps/web/src/lib/notifications/send-push.ts`
  - Used in: DriveCommand mobile app for delivery/driver alerts

**Maps & Geolocation:**
- **Google Maps API** (optional) - Address autocomplete, geocoding
  - API Key: `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY`
  - Status: Optional (not fully integrated in current codebase)

**Mapping & Location Display:**
- **Leaflet + React Leaflet** - Interactive maps for fleet tracking
  - SDK: `leaflet`, `react-leaflet`, `react-leaflet-cluster`
  - No API key required (uses OpenStreetMap by default)
  - Used in: DriveCommand web for driver location visualization

## Data Storage

**Databases:**

**Primary Database (DriveCommand Web, LineUp):**
- **PostgreSQL** (via Supabase)
  - Provider: Supabase (managed PostgreSQL)
  - Connection: `DATABASE_URL` (Supabase connection string)
  - Client: Prisma ORM (`@prisma/client` with `@prisma/adapter-pg`)
  - Connection pooling: Prisma handles with PgBouncer
  - URL: `NEXT_PUBLIC_SUPABASE_URL`
  - Schema: `apps/web/prisma/schema.prisma` (Prisma models)
  - Multi-tenancy: All tables include `tenant_id` column
  - RLS: Row-level security policies in Supabase via `rls-tenant-integrations.sql`
  - Migrations: `prisma/migrations/` directory (auto-generated by Prisma)

**Real-Time Database (LineUp):**
- **Supabase PostgreSQL** with LISTEN/NOTIFY
  - Enabled via `@supabase/supabase-js` subscriptions
  - Real-time updates for collaborative editing (Phases, Requirements, etc.)

**Caching:**
- **Upstash Redis** - Distributed rate limiting
  - SDK: `@upstash/redis`, `@upstash/ratelimit`
  - Configuration:
    - `UPSTASH_REDIS_REST_URL` - REST endpoint
    - `UPSTASH_REDIS_REST_TOKEN` - Auth token
  - Client: `apps/web/src/lib/rate-limit.ts`
  - Purpose: Rate limit API endpoints (e.g., form submissions, file uploads)
  - Alternative: Not required for development (can use in-memory fallback)

**Local Mobile Storage:**
- **react-native-mmkv** - Fast, encrypted local storage (DriveCommand Mobile)
  - Used for: Offline data cache, session tokens, user preferences

**File Storage:**
- **S3/R2 buckets** (see "File Storage" under APIs above)

## Authentication & Identity

**Auth Provider:**
- **Supabase Auth** - Passwordless or email/password, OAuth support
  - Implementation: JWT tokens in HTTP-only cookies
  - Token refresh: Via Supabase session management
  - Multi-role RBAC:
    - `sys_admin` - Global superuser (access to `/admin` portal)
    - `org_admin` - Tenant owner (manage team, billing)
    - `org_user` - Standard employee (access to assigned tenant's data)
    - `client_user` - Portal user (restricted to specific client)
  - Role source: `user_profiles.role` table field checked via Supabase RLS
  - Custom claims: `tenant_id` embedded in JWT via Supabase custom claims

**Session Management:**
- Cookies: HTTP-only, Secure, SameSite
- Middleware: `apps/web/src/lib/supabase/middleware.ts` refreshes tokens on each request
- Client: `@supabase/ssr` handles SSR session hydration

**Password Security:**
- Supabase Auth handles password hashing (bcrypt internally)
- Admin password: `ADMIN_SECRET_KEY` for super-admin `/admin/login` portal
- Password reset: Via Supabase email links (Resend)

## Monitoring & Observability

**Error Tracking:**
- **Sentry** - Error monitoring, performance tracking
  - Web: `@sentry/nextjs` 10.46.0
  - Mobile: `@sentry/react-native` 7.11.0
  - Configuration:
    - `NEXT_PUBLIC_SENTRY_DSN` - Browser/client-side DSN (public)
    - `SENTRY_DSN` - Server-side DSN
    - `SENTRY_AUTH_TOKEN` - CLI token for source map uploads
  - Config: `apps/web/next.config.ts` wraps Next.js with `withSentryConfig`
  - Auto-captures: Unhandled errors, performance metrics, replay traces

**Logs:**
- **Server Logs** - Console logs via Node.js `console` (captured by Vercel/hosting)
- **Browser Logs** - JavaScript console (visible in browser DevTools)
- **Structured Logging** - Manual logging in server actions / API routes for key events
- No external logging service currently integrated (Sentry captures errors)

**Analytics:**
- Not integrated (could use PostHog, Segment, etc.)

## Environment Configuration

**Required env vars (All Projects):**
- `VITE_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_URL` - Supabase project URL
- `VITE_SUPABASE_ANON_KEY` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` - Public key (safe for browser)

**Required env vars (DriveCommand Web):**
- `DATABASE_URL` - PostgreSQL connection string
- `NEXT_PUBLIC_APP_URL` - App URL for email links
- `SUPABASE_SERVICE_ROLE_KEY` - Server-side Supabase operations
- `RESEND_API_KEY` - Email service
- `RESEND_FROM_EMAIL` - Sender email address
- `UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN` - Rate limiting
- `ADMIN_SECRET_KEY` - Super-admin portal
- `CRON_SECRET` - Vercel cron protection
- `AUTH_SECRET` - JWT signing (if using NextAuth)

**Optional env vars (DriveCommand Web):**
- `ANTHROPIC_API_KEY` - Claude API (document AI)
- `S3_ENDPOINT`, `S3_REGION`, `S3_ACCESS_KEY_ID`, `S3_SECRET_ACCESS_KEY`, `S3_BUCKET` - File storage
- `NEXT_PUBLIC_SENTRY_DSN`, `SENTRY_DSN`, `SENTRY_AUTH_TOKEN` - Error tracking
- `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` - Address autocomplete

**Secrets location:**
- **Development**: `.env.local` (git-ignored) — copy from `.env.example`
- **Production (Vercel)**: Environment Variables in Vercel dashboard (Settings → Environment Variables)
- **CI/CD**: Repository secrets (GitHub Actions, etc.)

## Data Security & Compliance

**Row-Level Security (RLS):**
- Supabase RLS policies enforce `tenant_id` isolation
- File: `apps/web/prisma/rls-tenant-integrations.sql`
- Every query from browser checked against JWT tenant_id

**Encryption:**
- Database: Encrypted in transit (TLS), at rest (Supabase managed)
- Files: Encrypted at rest (S3/R2 default), in transit (HTTPS)
- Local storage: Encrypted by default (`react-native-mmkv`)

**API Rate Limiting:**
- Upstash Redis: Prevent abuse on form submissions, file uploads
- Default: Applied to public routes (login, signup, etc.)
